Helix-Bio

Domain-specialized biomedical research agent — UniProt · PDB · AlphaFold · BLAST · ChEMBL · PubMed native.

Design target: faithfulness-gated outputs matching GPT-Rosalind's BixBench tier (0.751 pass rate reported) without requiring a new model weight — by combining a strong generalist with typed MCP tools for authoritative databases, RadAgent-style inspectable traces, and an ontology-binding structural constraint on outputs.

What Helix-Bio is

A tool-using biomedical research agent. The pipeline mirrors Atlas-Research (intake → plan → retrieve → synthesize → verify) but specializes the retrieval layer to authoritative life-sciences sources reached through typed MCP tools, and upgrades the faithfulness gate with ontology-identifier binding as a structural output constraint.

Niche

Target discovery, variant reasoning, protein analysis, pathway landscape, compound druggability — decision support for bench scientists.

Not

A clinical decision system; a replacement for wet lab; a GxP-validated pipeline.

Status

Design specification. Scaffold-quality. Not implemented.

Novel contributions (not present in cited systems)

1. Ontology-identifier-bound outputs as a structural constraint

Every gene/protein/drug mention must bind to a canonical identifier (UniProt AC, HGNC, MeSH, ChEMBL) validated by the gate against the trace. This tightens RadAgent's faithfulness pattern by fusing it with Atlas-Research's claim-evidence binding. Cited systems treat identifiers as metadata; Helix-Bio treats them as a hard constraint the output must satisfy before emission.

2. Neuro-symbolic residue-level verifier

Structural claims ("R273 in TP53 DBD H-bonds with DNA phosphate") route to a geometry checker against cited PDB coordinates rather than LLM self-critique. Direct application of the Marcus & Belle neuro-symbolic design pattern to structural biology.

3. ReBalance confidence-steering on the retrieval DAG

Easy ID lookups run shallow; hard compositional reasoning steers deeper via the training-free ReBalance mechanism. First documented application of reasoning-depth control to biomedical RAG.

4. Dual-use safety as a harness perimeter

Deterministic deny-list (BSAT, Schedule-I routes, gain-of-function uplift queries) + institutional-review routing lane + content classifier, composed at the harness layer — not as a trailing moderation pass. See block 07.

Why this stack, block by block

Why RadAgent-style inspectable traces?

Generalist agents emit opinions; clinicians and bench scientists need audit trails. RadAgent showed +24.7 robustness and introduced faithfulness as a capability by exposing the reasoning steps. Helix-Bio pays the cost of that visibility because biomedical outputs are high-stakes.

Why identifier-bound outputs?

Generic RAG pipelines cite URLs and abstract chunks. That resolves "where I read it" but not "which specific entity". An output mentioning "KRAS" without binding to UniProt:P01116 is unsafe in a target-discovery context where the wrong paralog is a real risk.

Why a generalist model rather than training our own?

GPT-Rosalind's advantage is partly from 50 workflow-trained examples. Helix-Bio's bet: equivalent domain quality comes from tool fidelity + evaluation discipline, not from model weights. Same-tier output without the training cost.

Why ReBalance here?

Biomedical RAG has a bimodal reasoning profile: most tasks are simple ID lookups (shallow); some are genuinely hard composition (variant × pathway × drug). Static reasoning depth wastes compute on easy and under-thinks on hard. ReBalance is the training-free mechanism for per-call depth control.
